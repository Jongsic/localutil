PWA Checker

Audit PWA install readiness from page HTML and manifest JSON — checklist, icons, install preview

Page source & manifest
Browsers can't fetch other sites' files from a page (CORS/CSP), so paste the page HTML and the manifest JSON. The URL is optional — it resolves relative paths, enables icon previews, and checks HTTPS.
Install readiness — what to fix, what is done
Every install requirement and recommendation is listed, including what's missing. ✕ blocks install  ·  ! should fix  ·  ○ optional, not set  ·  ✓ done
Manifest icons
Previews load straight from the icon URLs (needs the page URL for relative paths). The dashed circle on maskable icons is the safe zone — everything important must fit inside it.

Service worker — verify it live
Registration can only be confirmed on the running site. Open the site in a tab, then run this in its DevTools console:
navigator.serviceWorker.getRegistrations().then(rs => console.log(rs.length ? rs.map(r => r.scope + ' → ' + (r.active ? r.active.scriptURL : '(no active worker)')) : 'no service worker registrations'))
chrome:// pages can't be opened from a link — paste the copied address into a new tab. Or use DevTools → Application → Service Workers / Manifest, or a Lighthouse audit.
